# Стиль кода Ясного

Этот документ описывает единый стиль записи кода на Ясном.
Стиль — не вопрос вкуса, а **общий канон**, чтобы любой код в
экосистеме читался одинаково. Команда `yasny формат` приводит
файлы к этому стилю автоматически.

Главный принцип: **меньше когнитивной нагрузки на читателя**.
Если правило снижает нагрузку — оно есть. Если правило её не
снижает — его нет.

## Отступы

- Один уровень = **4 пробела**.
- Никогда не tab.
- Закрывающее `конец` стоит на том же уровне, что открывающее
  ключевое слово (`функция`, `если`, `для`, `пока`, `класс`,
  `попытка`).

```
функция плюс(a, b)
    вернуть a + b
конец
```

## Пробелы

| Где | Пример | Почему |
|---|---|---|
| Один пробел вокруг бинарных операторов | `a + b`, `x == y`, `n % 2` | Глаз отделяет операнды от оператора |
| Без пробела вокруг унарных | `-x`, `не x`, `...args` | Унарный связан с операндом |
| Один пробел после запятой | `[1, 2, 3]`, `f(a, b)` | Разделение элементов |
| Один пробел после двоеточия | `если x: y`, `когда 1: "Пн"` | Чёткое отделение тела от условия |
| Один пробел после ключевых слов | `если x`, `для i от 1 до 10` | Слово отделено от выражения |
| Без пробелов внутри скобок | `f(2, 3)`, `[1, 2]`, `{a: 1}` | Содержимое прижато к скобкам |
| Без пробелов вокруг точки | `это.имя`, `мат.пи` | Точка — связывание |
| Один пробел вокруг `=` | `конст x = 10` | Присваивание читается как «равно» |
| Один пробел в стрелке лямбды | `x => x * 2` | Стрелка отделена от тела |
| Один пробел вокруг `\|>` | `данные \|> фильтр(...)` | Пайп отделён от шагов |

## Переменные

- `конст` — по умолчанию (см. принцип 4 — иммутабельность).
- `перем` — только когда переменная реально меняется.

```
конст имя = "Аня"
конст возраст = 25
перем счётчик = 0
```

## Имена

| Что | Стиль | Пример |
|---|---|---|
| Переменные, функции | `строчные_через_подчёркивание` | `сумма_цифр`, `найти_первое` |
| Классы, перечисления | `СБольшойБуквыКамел` | `Точка`, `СписокЗадач` |
| Константы | строчные, как переменные | `пи`, `золотое_сечение` |
| Предикаты (`да`/`нет`) | заканчиваются на `?` | `простое?`, `пустой?`, `чётное?` |
| Внутренние имена | начинаются с `_` | `_вспомогательная_функция` |

Не пишем константы заглавными (`ПИ`, `ЗОЛОТОЕ_СЕЧЕНИЕ`) — это «крик»,
шумит при чтении. Малые буквы спокойнее.

## Кавычки

- Только двойные `"..."` для строк.
- Двойные кавычки поддерживают интерполяцию `{...}` — это полезно
  и часто нужно.
- Одинарные `'...'` существуют, но их не используем — без причины
  не вводим разнообразия там, где можно одинаково.

```
конст приветствие = "Привет, {имя}!"
```

## Комментарии

- Только `#`, никаких `//` или `/* */`.
- На отдельной строке **перед** кодом, который описывают.
- Пишутся как обычные предложения с заглавной буквы и точкой в конце.
- После `#` всегда один пробел.

```
# Считает n-е число Фибоначчи через рекурсию.
функция фиб(n)
    если n < 2: вернуть n
    вернуть фиб(n - 1) + фиб(n - 2)
конец
```

## Управление потоком

### Условия

```
если возраст >= 18
    вывод("Взрослый")
иначеесли возраст >= 14
    вывод("Подросток")
иначе
    вывод("Ребёнок")
конец
```

Короткая форма для одного выражения — допустима и предпочтительна:

```
если n <= 1: вернуть 1
если n < 0: бросить "отрицательное число"
```

### Циклы

```
для i от 1 до 10
    тело
конец

для i от 0 до 10 по 2
    тело
конец

для x в массив
    тело
конец

для индекс, x в массив
    тело
конец

пока условие
    тело
конец
```

### Сопоставление с образцом

```
совпадение значение
    когда 1: "один"
    когда 2: "два"
    иначе: "неизвестно"
конец
```

Каждое `когда` с отступом 4. Один пробел после двоеточия.

## Функции

### Блочная форма

```
функция факториал(n)
    если n <= 1: вернуть 1
    вернуть n * факториал(n - 1)
конец
```

### Inline-форма (одно выражение)

```
функция квадрат(x): x * x
функция чётное?(n): n % 2 == 0
```

Используется только когда тело функции — одно короткое выражение.
Сохраняет короткое в одну строку — это снижает нагрузку.

### Параметры по умолчанию

```
функция степень(основание, показатель = 2)
    вернуть основание * основание
конец
```

### Переменное число параметров

```
функция максимум(...числа)
    тело
конец
```

### Лямбды

- `x => x * 2` — один параметр без скобок.
- `(a, b) => a + b` — несколько параметров со скобками.

## Классы

```
класс Точка
    функция создать(x, y)
        это.x = x
        это.y = y
    конец

    функция длина()
        вернуть корень(это.x * это.x + это.y * это.y)
    конец
конец
```

- Между методами — **1 пустая строка**.
- Конструктор — `создать`, всегда первый метод.
- Поле инстанса через `это.имя`.
- Родительский метод через `родитель.метод()`.

```
класс Кот наследует Животное
    функция голос()
        вернуть "{это.имя}: мяу"
    конец
конец
```

## Литералы

### Массивы

Однострочно, если короткие:

```
конст числа = [1, 2, 3, 4, 5]
```

Многострочно, если длинные или со структурами. **С trailing comma**:

```
конст команда = [
    {"имя": "Аня", "возраст": 22},
    {"имя": "Боря", "возраст": 25},
]
```

### Словари

То же правило. Однострочно:

```
конст человек = {"имя": "Иван", "возраст": 30}
```

Многострочно:

```
конст человек = {
    "имя": "Иван",
    "возраст": 30,
    "город": "Казань",
}
```

Trailing comma в многострочных — **обязательна**: это минимизирует
diff в git при добавлении нового элемента.

## Модули

### Импорты

В начале файла, перед основным кодом:

```
импорт м из "матем"
импорт с из "строки"
импорт время из "время"

вывод(м.факториал(5))
```

Между блоком импортов и кодом — **1 пустая строка**.

Порядок импортов **сохраняется как написал автор** — не сортируется
автоматически. Авторы часто группируют импорты по смыслу.

### Экспорты

```
экспорт конст пи = 3.141592653589793

экспорт функция квадрат(x): x * x
```

## Декораторы

На отдельной строке непосредственно перед функцией:

```
@логировать
функция квадрат(x): x * x

@логировать
@кешировать
функция дорогое_вычисление(n)
    тело
конец
```

## Pipeline

Многострочный pipeline с `|>` в начале продолжающей строки и
отступом 4:

```
конст результат = числа
    |> фильтр(x => x % 2 == 0)
    |> преобразовать(x => x * x)
    |> свернуть((s, x) => s + x, 0)
```

Однострочный — допустим, если короткий:

```
конст квадраты = числа |> преобразовать(x => x * x)
```

## Пустые строки

- **Одна** пустая строка между функциями верхнего уровня.
- **Одна** пустая строка между методами класса.
- **Одна** пустая строка между блоком импортов и кодом.
- **Ноль** пустых строк перед `конец`, `иначе`, `иначеесли`.
- **Ноль** пустых строк в начале и конце файла.
- **Ноль или одна** пустая строка для смыслового разделения внутри
  тела функции — две и больше **схлопываются в одну**.

## Длина строк

- Рекомендуемая длина строки — **до 100 символов**.
- Форматер не переносит длинные строки автоматически — это
  ломает намеренное форматирование. Длинные строки помечаются
  предупреждением (`yasny формат --проверить`).

## Точки с запятой

**Не используются. Никогда.**

Перенос строки = конец инструкции. Это однозначно и не требует
дополнительного символа.

## Запрещённые формы

Эти формы удалены из языка в версии 0.5 и не должны встречаться
в новом коде:

- `пусть` — используйте `конст` или `перем`.
- `var` — используйте `перем`.
- `процедура` — используйте `функция`.
- `возврат` — используйте `вернуть`.
- `истина`, `ложь` — используйте `да`, `нет`.
- `пусто` — используйте `ничего`.
- `//` как комментарий — используйте `#`.

## Точно-канонические примеры

Хорошо:

```
# Считает количество простых чисел до n.
функция простых_до(n)
    перем счёт = 0
    для i от 2 до n
        если простое?(i)
            счёт += 1
        конец
    конец
    вернуть счёт
конец

класс Очередь
    функция создать()
        это.элементы = []
    конец

    функция добавить(x)
        добавить(это.элементы, x)
    конец

    функция взять()
        если длина(это.элементы) == 0: вернуть ничего
        конст первый = это.элементы[0]
        это.элементы = это.элементы[1:]
        вернуть первый
    конец
конец
```

Плохо (форматер исправит):

```
функция плохо(  x,y){               # лишние пробелы, скобки в стиле C
    вернуть x+y;                    # нет пробелов, точка с запятой
}                                   # вместо `конец`



# Две лишних пустых строки сверху.
конст ПИ=3.14                       # верхний регистр для константы, нет пробелов
```

## Что делает форматер

1. Приводит отступы к 4 пробелам.
2. Расставляет пробелы вокруг операторов и после знаков пунктуации.
3. Удаляет точки с запятой.
4. Схлопывает множественные пустые строки.
5. Ставит trailing comma в многострочных литералах.
6. Удаляет trailing whitespace в конце строк.
7. Гарантирует одну `\n` в конце файла.
8. Не сортирует импорты, не сокращает длинные строки, не меняет
   смысловое разбиение.

## Что форматер НЕ делает

- Не сортирует импорты — порядок несёт смысл.
- Не переносит длинные строки — авто-перенос ломает намеренное
  оформление.
- Не объединяет inline и блочные формы — автор сам выбирает.
- Не меняет имена переменных, функций, классов — это смысл, не стиль.
- Не меняет структуру кода (циклы в цикл while и т.п.).

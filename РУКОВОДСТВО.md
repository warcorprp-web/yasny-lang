## Работа с числами

### Деление, остаток и целочисленное деление

```
вывод(10 / 3)      # 3.333...   обычное деление (всегда float если не делится нацело)
вывод(10 // 3)     # 3          целочисленное деление
вывод(10 % 3)      # 1          остаток
вывод(10.5 % 3)    # 1.5        остаток для float тоже работает
вывод(10 / 2)      # 5          точное деление возвращает int
```

### Сумма от 1 до N

```
функция сумма_до(n)
    перем итог = 0
    для i от 1 до n: итог += i
    вернуть итог
конец

вывод(сумма_до(100))    # 5050
```

### Чётное или нечётное

```
функция чётное?(n): n % 2 == 0

вывод(чётное?(4))    # да
вывод(чётное?(7))    # нет
```

### Среднее арифметическое

```
функция среднее(числа)
    если длина(числа) == 0: вернуть 0
    вернуть сумма(числа) / длина(числа)
конец

вывод(среднее([10, 20, 30, 40]))    # 25
```

### Округление до N знаков

```
функция округлить_до(x, знаков)
    конст множитель = степень(10, знаков)
    вернуть округл(x * множитель) / множитель
конец

вывод(округлить_до(3.14159, 2))    # 3.14
```

### Случайное число в диапазоне

```
вывод(случайное.целое(1, 100))     # целое 1..100
вывод(случайное.число())           # дробное 0..1
```

## Работа со строками

### Индексация и срезы

Строки индексируются по символам (не байтам), кириллица работает
корректно:

```
конст с = "привет"
вывод(с[0])           # п
вывод(с[5])           # т
вывод(с[-1])          # т (отрицательные индексы — с конца)
вывод(длина(с))       # 6 символов

вывод(подстрока(с, 0, 3))   # при
вывод(подстрока(с, 3, 6))   # вет
```

### Сравнение строк

Строки сравниваются лексикографически:

```
вывод("а" < "б")            # да
вывод("я" > "а")            # да
вывод("apple" < "banana")   # да
вывод("abc" == "abc")       # да
```

### Перевернуть строку

```
функция перевернуть(s)
    перем результат = ""
    для i от длина(s) - 1 до 0 по -1
        результат += s[i]
    конец
    вернуть результат
конец

вывод(перевернуть("Привет"))    # тевирП
```

### Подсчёт слов

```
функция количество_слов(текст)
    вернуть длина(разделить(обрезать(текст), " "))
конец

вывод(количество_слов("Это предложение из пяти слов"))    # 5
```

### Замена нескольких подстрок

```
функция заменить_многое(текст, замены)
    перем результат = текст
    для пара в замены
        результат = заменить(результат, пара[0], пара[1])
    конец
    вернуть результат
конец

вывод(заменить_многое("Привет, мир", [["Привет", "Hello"], ["мир", "world"]]))
```

### Проверка, что строка — число

```
функция это_число(s)
    попытка
        число(s)
        вернуть да
    поймать _
        вернуть нет
    конец
конец

вывод(это_число("42"))      # да
вывод(это_число("abc"))     # нет
```

## Списки и словари

### Уникальные элементы с сохранением порядка

```
конст исходный = [1, 2, 3, 2, 1, 4, 3]
вывод(уникальные(исходный))    # [1, 2, 3, 4]
```

### Сортировка по полю

```
конст люди = [
    {"имя": "Аня", "возраст": 25},
    {"имя": "Боря", "возраст": 22},
    {"имя": "Вася", "возраст": 30},
]

конст по_возрасту = сортировать(люди, (a, b) => a["возраст"] - b["возраст"])

для ч в по_возрасту
    вывод("{ч["имя"]}: {ч["возраст"]}")
конец
```

### Группировка

```
функция сгруппировать(массив, по_ключу)
    перем группы = {}
    для элемент в массив
        конст ключ = по_ключу(элемент)
        если не содержит(ключи(группы), ключ)
            группы[ключ] = []
        конец
        добавить(группы[ключ], элемент)
    конец
    вернуть группы
конец

конст числа = [1, 2, 3, 4, 5, 6]
конст по_чётности = сгруппировать(числа, x => если x % 2 == 0: "чётные" иначе: "нечётные")
вывод(по_чётности)
```

### Поиск максимума по полю

```
функция максимум_по(массив, по_ключу)
    если длина(массив) == 0: вернуть ничего
    перем лучший = массив[0]
    для элемент в массив
        если по_ключу(элемент) > по_ключу(лучший)
            лучший = элемент
        конец
    конец
    вернуть лучший
конец

конст люди = [{"имя": "А", "балл": 5}, {"имя": "Б", "балл": 9}, {"имя": "В", "балл": 7}]
вывод(максимум_по(люди, ч => ч["балл"]))
```

### Объединение словарей

```
функция объединить_словари(a, b)
    перем итог = {}
    для к в ключи(a): итог[к] = a[к]
    для к в ключи(b): итог[к] = b[к]
    вернуть итог
конец
```

## Управление потоком

### Литерал ничего

`ничего` — null-значение языка. Используется как «значение
отсутствует»:

```
функция найти(числа, цель)
    для n в числа
        если n == цель: вернуть n
    конец
    вернуть ничего
конец

конст результат = найти([1, 2, 3], 5)
если результат == ничего
    вывод("не найдено")
иначе
    вывод("найдено: " + строка(результат))
конец
```

Удобный шорткат — `??` (nullish coalescing): возвращает левое
значение, если оно не `ничего`, иначе правое:

```
конст имя = пользователь?.имя ?? "Аноним"
```

### Ранний выход из функции

```
функция найти_первое_отрицательное(числа)
    для n в числа
        если n < 0: вернуть n
    конец
    вернуть ничего
конец
```

### Прерывание цикла

```
для i от 1 до 1000
    если i * i > 100: прервать
    вывод(i)
конец
```

### Пропуск итерации

```
для n в [1, 2, 3, 4, 5, 6]
    если n % 2 == 0: продолжить
    вывод(n)    # 1, 3, 5
конец
```

### Цикл с условием выхода в конце

В Ясном нет `do-while`, но можно сделать так:

```
перем готово = нет
пока не готово
    конст ввод_пользователя = ввод("Введите 'выход' для выхода: ")
    если ввод_пользователя == "выход"
        готово = да
    иначе
        вывод("Вы ввели: {ввод_пользователя}")
    конец
конец
```

## Функции

### Функции-предикаты со знаком вопроса

Имена функций, возвращающих `да`/`нет`, могут заканчиваться на `?` —
это идиоматичный способ обозначить предикат:

```
функция чётное?(n): n % 2 == 0
функция пустой?(массив): длина(массив) == 0
функция простое?(n)
    если n < 2: вернуть нет
    если n < 4: вернуть да
    если n % 2 == 0: вернуть нет
    перем д = 3
    пока д * д <= n
        если n % д == 0: вернуть нет
        д = д + 2
    конец
    вернуть да
конец

вывод(чётное?(4))      # да
вывод(простое?(13))    # да
```

Знак `?` не конфликтует ни с тернарным оператором (`x ? a : b`),
ни с optional chaining (`obj?.поле`).

### Функция с переменным числом аргументов

```
функция максимум_всех(...числа)
    если длина(числа) == 0: бросить "нужен хотя бы один аргумент"
    перем m = числа[0]
    для n в числа
        если n > m: m = n
    конец
    вернуть m
конец

вывод(максимум_всех(3, 7, 1, 9, 4))    # 9
```

### Функция, возвращающая несколько значений

```
функция деление_с_остатком(a, b)
    вернуть a / b, a % b
конец

конст ч, о = деление_с_остатком(17, 5)
вывод("17 / 5 = {ч}, остаток {о}")
```

Множественный возврат работает только в блочной форме функции
(с `вернуть` и `конец`), не в inline-форме через двоеточие.

### Функция как параметр

```
функция применить_дважды(f, x)
    вернуть f(f(x))
конец

вывод(применить_дважды(x => x + 1, 5))    # 7
```

### Замыкание (функция, помнящая значения)

```
функция счётчик_от(начало)
    перем текущий = начало
    вернуть () => {
        текущий += 1
        вернуть текущий
    }
конец
```

## Классы и объекты

### Простой класс с методами

```
класс Прямоугольник
    функция создать(ширина, высота)
        это.ширина = ширина
        это.высота = высота
    конец

    функция площадь()
        вернуть это.ширина * это.высота
    конец

    функция периметр()
        вернуть 2 * (это.ширина + это.высота)
    конец
конец

конст p = Прямоугольник(5, 3)
вывод(p.площадь())       # 15
вывод(p.периметр())      # 16
```

### Наследование

```
класс Фигура
    функция создать(цвет)
        это.цвет = цвет
    конец

    функция описание()
        вернуть "Фигура цвета {это.цвет}"
    конец
конец

класс Круг наследует Фигура
    функция создать(цвет, радиус)
        родитель.создать(цвет)
        это.радиус = радиус
    конец

    функция площадь()
        вернуть мат.пи * это.радиус * это.радиус
    конец
конец

конст c = Круг("красный", 5)
вывод(c.описание())     # Фигура цвета красный
вывод(c.площадь())      # 78.5...
```

### Статичный счётчик экземпляров

```
класс Пользователь
    функция создать(имя)
        это.имя = имя
    конец
конец

# Статичность пока эмулируется через внешнюю переменную
перем количество_пользователей = 0

функция новый_пользователь(имя)
    количество_пользователей += 1
    вернуть Пользователь(имя)
конец
```

## Сеть и HTTP

### HTTP-клиент

```
# GET
конст ответ = http.получить("https://api.example.com/users/1")
вывод(ответ.статус)    # 200
вывод(ответ.тело)      # строка

# POST с JSON
конст ответ = http.пост("https://api.example.com/users", {"имя": "Аня", "возраст": 25})

# Все методы: получить, пост, положить, удалить, патч
# Третий аргумент — заголовки:
конст ответ = http.получить("https://api.example.com/data", ничего, {"Authorization": "Bearer token123"})
```

### HTTP-сервер (приложение с routing)

```
конст app = http.приложение()

# Middleware (вызывается перед каждым обработчиком)
app.использовать(функция(запрос)
    вывод(запрос.метод + " " + запрос.путь)
конец)

# Маршруты
app.получить("/", функция(запрос)
    вернуть "<h1>Главная</h1>"
конец)

app.получить("/api/пользователи", функция(запрос)
    конст имя = запрос.параметры["имя"] ?? "Мир"
    вернуть {"сообщение": "Привет, " + имя + "!"}
конец)

app.пост("/api/данные", функция(запрос)
    конст данные = запрос.json  # автопарсинг JSON body
    вернуть {"статус": 201, "тело": {"получено": данные}}
конец)

# Статические файлы
app.статика("/static", "./public")

app.запустить(8080)
```

### Объект запроса

Обработчик получает hash с полями:
- `метод` — GET, POST, PUT, DELETE, PATCH
- `путь` — /api/users
- `url` — полный URL с query string
- `параметры` — query params как hash (`?имя=Аня` → `запрос.параметры["имя"]`)
- `заголовки` — hash заголовков (ключи в нижнем регистре)
- `тело` — строка тела запроса
- `json` — автоматически распарсенный JSON (если Content-Type: application/json)

### Объект ответа

Обработчик может вернуть:
- **строку** → `text/html`, статус 200
- **hash/array** (без поля `статус`) → автоматически JSON, статус 200
- **hash с полем `статус`** → полный контроль:
  ```
  вернуть {
      "статус": 201,
      "тело": {"id": 1, "создано": да},
      "заголовки": {"X-Custom": "значение"},
      "тип": "application/json; charset=utf-8"
  }
  ```
- **ничего** → статус 204 (No Content)

### Простой сервер (обратная совместимость)

```
http.сервер(8080, функция(запрос)
    вернуть "Привет с " + запрос.путь
конец)
```

## Шаблонизатор

Модуль `шаблон` генерирует HTML (или любой текст) из шаблонов с данными.

### Синтаксис шаблонов

```
{{выражение}}                — вставка значения
{{если условие}}...{{конец}} — условие
{{если x}}...{{иначе}}...{{конец}}
{{для элемент в массив}}...{{конец}} — цикл
{{включить "файл.html"}}    — вставка другого файла
```

### Использование

```
# Из строки
конст html = шаблон.рендер('<h1>{{имя}}</h1><p>{{возраст}} лет</p>', {
    "имя": "Анна",
    "возраст": 25
})

# Из файла
конст страница = шаблон.файл("шаблоны/главная.html", {
    "заголовок": "Мой сайт",
    "товары": [{"название": "Хлеб", "цена": 50}]
})
```

### Пример шаблона

```
<ul>
{{для товар в товары}}
  <li>{{товар.название}} — {{товар.цена}} руб.</li>
{{конец}}
</ul>

{{если пользователь}}
  <p>Привет, {{пользователь.имя}}!</p>
{{иначе}}
  <p><a href="/войти">Войти</a></p>
{{конец}}
```

### Экранирование HTML

```
конст безопасный = шаблон.экранировать("<script>alert('xss')</script>")
# &lt;script&gt;alert(&#39;xss&#39;)&lt;/script&gt;
```

### Form data в HTTP-сервере

При POST с `Content-Type: application/x-www-form-urlencoded` данные
формы автоматически парсятся в `запрос.форма`:

```
app.пост("/регистрация", функция(запрос)
    конст имя = запрос.форма["имя"]
    конст email = запрос.форма["email"]
    # ...
конец)
```

## Регулярные выражения

### Проверка по шаблону

```
вывод(регвыр.совпадает("user@mail.ru", "[a-z]+@[a-z]+\\.[a-z]+"))
```

### Найти все совпадения

```
конст числа = регвыр.найти("В 2024 году было 365 дней", "[0-9]+")
вывод(числа)    # [2024, 365]
```

### Замена

```
конст текст = регвыр.заменить(
    "Привет, мир!",
    "мир",
    "Ясный"
)
вывод(текст)    # Привет, Ясный!
```

## Базы данных (SQLite + PostgreSQL)

Модуль `бд` предоставляет полноценную работу с реляционными базами
данных. Поддерживаются SQLite (встроенный, файловый) и PostgreSQL
(сетевой). API одинаковый — код работает с обеими БД без изменений.

### Подключение

```
импорт бд из "бд"

# SQLite — файл или память
конст соед = бд.открыть(":memory:")
конст соед = бд.открыть("данные.db")

# PostgreSQL — строка подключения
конст соед = бд.подключить("postgres://user:pass@localhost:5432/mydb?sslmode=disable")
```

### Основные операции

```
# Создание таблицы
соед.выполнить("
    CREATE TABLE пользователи (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        имя TEXT NOT NULL,
        email TEXT UNIQUE,
        возраст INTEGER
    )
")

# Вставка (ВСЕГДА используйте ? для значений — защита от SQL-инъекций)
соед.выполнить("INSERT INTO пользователи (имя, email, возраст) VALUES (?, ?, ?)", "Анна", "anna@mail.ru", 25)

# Запрос — возвращает массив hash-ей
конст люди = соед.запрос("SELECT * FROM пользователи WHERE возраст > ?", 20)
для п в люди
    вывод(п.имя + " — " + строка(п.возраст))
конец

# Одна строка (или ничего если не найдена)
конст один = соед.строка("SELECT * FROM пользователи WHERE id = ?", 1)

# Скалярное значение (COUNT, SUM, AVG, MAX, MIN)
конст всего = соед.значение("SELECT COUNT(*) FROM пользователи")
```

### Prepared Statements

Для многократного выполнения одного запроса с разными параметрами.
Быстрее чем повторные вызовы `выполнить` — SQL компилируется один раз.

```
конст stmt = соед.подготовить("INSERT INTO пользователи (имя, возраст) VALUES (?, ?)")
stmt.выполнить("Иван", 30)
stmt.выполнить("Олег", 22)
stmt.выполнить("Маша", 28)
stmt.закрыть()

# Для SELECT:
конст поиск = соед.подготовить("SELECT имя FROM пользователи WHERE возраст > ?")
конст результат = поиск.запрос(25)
поиск.закрыть()
```

### Пакетная вставка

Вставка множества строк одной командой:

```
соед.вставить_много("пользователи", ["имя", "email", "возраст"], [
    ["Анна", "anna@mail.ru", 25],
    ["Иван", "ivan@mail.ru", 30],
    ["Олег", "oleg@mail.ru", 22],
])
```

### Транзакции

Группа операций выполняется атомарно: либо все, либо ни одна.
Если функция бросает ошибку — все изменения откатываются.

```
соед.транзакция(функция(тх)
    тх.выполнить("INSERT INTO заказы (сумма) VALUES (?)", 1500)
    тх.выполнить("UPDATE баланс SET сумма = сумма - ? WHERE id = ?", 1500, 1)
    # Если здесь ошибка — обе операции откатятся
конец)
```

### Миграции

Идемпотентное обновление схемы. Каждая миграция применяется ровно
один раз, повторный вызов пропускается.

```
соед.мигрировать(1, "CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)")
соед.мигрировать(2, "ALTER TABLE users ADD COLUMN email TEXT")
соед.мигрировать(3, "CREATE INDEX idx_email ON users(email)")

вывод(соед.версия())  # 3
```

### Интроспекция

```
вывод(соед.таблицы())  # ["users", "orders"]

для к в соед.колонки("users")
    вывод(к.имя + " " + к.тип)  # id INTEGER, name TEXT, ...
конец
```

Каждая колонка содержит поля: `имя`, `тип`, `обязательное` (да/нет),
`первичный_ключ` (да/нет), `по_умолчанию` (значение или ничего).

## WebSocket

Модуль `вс` позволяет подключаться к WebSocket-серверам для получения
данных в реальном времени (биржи, чаты, уведомления).

### Подключение и чтение

```
конст сокет = вс.подключить("wss://stream.binance.com:9443/ws/btcusdt@trade")
сокет.таймаут(10)

# Получить 5 сообщений
сокет.слушать(5, функция(сообщение)
    конст данные = json.разобрать(сообщение)
    вывод("BTC: $" + данные["p"])
конец)

сокет.закрыть()
```

### Отправка сообщений

```
сокет.отправить("ping")
сокет.отправить_json({"action": "subscribe", "channel": "trades"})
```

### Блокирующее чтение

```
# Читать по одному сообщению
пока сокет.открыт?()
    конст msg = сокет.получить()
    если msg == ничего: прервать
    вывод(msg)
конец
```

### Прерывание слушания

Если обработчик в `слушать` возвращает `нет` — цикл прерывается:

```
сокет.слушать(1000, функция(msg)
    конст д = json.разобрать(msg)
    если д["цена"] > 100000
        вывод("Цель достигнута!")
        вернуть нет  # прекратить слушание
    конец
конец)
```

### WebSocket-сервер

```
вс.сервер(8080, функция(клиент)
    пока клиент.открыт?()
        конст msg = клиент.получить()
        если msg == ничего: прервать
        клиент.отправить("Эхо: " + msg)
    конец
конец)
```

Сервер принимает подключения на указанном порту. Для каждого клиента
вызывается обработчик с объектом `клиент` (те же методы: `отправить`,
`получить`, `закрыть`, `открыт?`).

## Пакеты

Ясный поддерживает работу с пакетами, опубликованными на GitHub.
Никакого центрального сервера нет — пакеты хранятся в обычных
git-репозиториях, а версии — в виде git-тегов. Есть [реестр коротких
имён](https://github.com/warcorprp-web/yasny-registry), куда любой
может добавить свой пакет через PR.

### Самый быстрый старт

```bash
# 1. Создать файл программы
echo 'импорт м из "матем"
вывод(м.факториал(5))' > программа.ya

# 2. Подключить пакет (создаст пакет.json автоматически)
yasny подключить матем

# 3. Запустить
yasny программа.ya
```

Вывод:

```
120
```

При первом `yasny подключить` в папке без `пакет.json` он создаётся
автоматически с разумными умолчаниями. Никакого отдельного `инит`
для одного скрипта не требуется.

### Создать большой проект явно

```
yasny инит мой_проект
cd мой_проект
```

Команда создаёт:
- `пакет.json` — манифест проекта (имя, версия, точка входа, зависимости)
- `главный.ya` — точка входа
- `.gitignore` — исключает папку `пакеты/`

### Подключение по короткому имени

```
yasny подключить матем
yasny подключить строки
```

Yasny скачает реестр имён с GitHub, найдёт URL, склонирует.

### Подключение по полному URL

Если вашего пакета нет в реестре или вы хотите конкретный fork/версию:

```
yasny подключить github.com/warcorprp-web/matematika_plus
yasny подключить github.com/warcorprp-web/matematika_plus@v0.1.0
yasny подключить github.com/warcorprp-web/matematika_plus@main
```

Версия указывается через `@` (тег или ветка). Без версии — последний
коммит главной ветки.

### Файлы проекта

После установки появятся:

- `пакеты/имя/` — содержимое пакета (можно глянуть исходники)
- `пакет.json` — список зависимостей с человекочитаемыми именами
- `пакет.lock` — точные коммиты каждой зависимости (для воспроизводимости)

`пакет.lock` фиксирует, что у вас установлено сейчас. Если коммитить
его в git — другой человек сделает `yasny подключить` и получит
**ровно тот же** набор зависимостей, версия в версию. Это решает
проблему «у меня работает, а у тебя нет».

### Поделиться проектом

```bash
# Автор
git push

# Получатель
git clone репозиторий
cd репозиторий
yasny подключить        # ставит всё из пакет.lock (или пакет.json)
yasny запустить
```

### Использование пакета в коде

```
импорт м из "матем"

вывод(м.факториал(5))
вывод(м.простые_до(20))
```

Резолвер импортов работает так:
1. Если путь начинается с `./` или `../` или содержит `.ya` — это
   локальный файл, ищется относительно файла-импортёра.
2. Иначе считается именем пакета: ищется `пакеты/имя/`, точка входа
   берётся из `пакеты/имя/пакет.json`.

### Установить всё из манифеста

После клонирования чужого проекта:

```
yasny подключить
```

Команда без аргументов ставит все пакеты, перечисленные в манифесте.
Если есть `пакет.lock` — использует точные коммиты из него.

### Другие команды

```
yasny список              # показать установленные пакеты
yasny удалить ИМЯ         # убрать из манифеста и пакеты/
yasny запустить           # запустить точка_входа из манифеста
yasny запустить файл.ya   # запустить указанный файл
```

### Опубликовать свой пакет

1. Создайте репозиторий на GitHub.
2. Положите туда `пакет.json` и `главный.ya` с `экспорт функция ...`.
3. Поставьте git-тег версии: `git tag v0.1.0 && git push origin v0.1.0`.
4. Поделитесь URL вида `github.com/имя/репо@v0.1.0`.

### Добавить пакет в реестр коротких имён

1. Опубликуйте пакет (см. выше).
2. Сделайте форк репозитория [yasny-registry](https://github.com/warcorprp-web/yasny-registry).
3. Добавьте запись в `registry.json`:
   ```json
   "ваш_пакет": {
     "url": "github.com/ваш_логин/имя_репо",
     "описание": "Что делает пакет"
   }
   ```
4. Откройте Pull Request.

После принятия PR ваш пакет станет доступен по короткому имени:
`yasny подключить ваш_пакет`.

## Форматирование кода

Команда `yasny формат` автоматически приводит код к единому стилю.
Канон описан в [СТИЛЬ.md](СТИЛЬ.md). Форматер аналогичен `gofmt` (Go),
`prettier` (JS), `black` (Python) — один общий стиль для всей экосистемы.

### Использование

```
yasny формат файл.ya       — вывести отформатированный код в stdout
yasny формат -в файл.ya    — переписать файл на месте
yasny формат -в .          — все .ya в текущей папке (рекурсивно)
yasny формат -в проект/    — все .ya в указанной папке
```

При рекурсивном проходе форматер пропускает папки `пакеты/`, `.git/`
и `node_modules/`.

### Что делает форматер

- Расставляет 4 пробела отступа
- Ставит пробелы вокруг операторов и после знаков пунктуации
- Удаляет точки с запятой и trailing whitespace
- Схлопывает множественные пустые строки в одну
- Добавляет trailing comma в многострочных литералах
- Гарантирует одну `\n` в конце файла
- Нормализует комментарии (один пробел после `#`)

### Что форматер НЕ трогает

- Имена переменных, функций, классов
- Структуру кода (циклы, условия)
- Импорты — порядок остаётся как написал автор
- Длинные строки — не переносит автоматически
- Inline и блочные формы — оставляет выбор автора:
  - `функция f(x): x * x` остаётся inline
  - `если x: вернуть y` остаётся inline
- Семантику строк — двойные/одинарные кавычки не меняет
- Многострочные литералы остаются многострочными

### Пример

До (грязное форматирование):

```
функция плюс(a,b)
   вернуть a+b
конец

конст товары=[
   "хлеб",   "молоко","сыр",
]
```

После `yasny формат`:

```
функция плюс(a, b)
    вернуть a + b
конец

конст товары = [
    "хлеб",
    "молоко",
    "сыр",
]
```

Программа после форматирования работает идентично — это гарантия
форматера (round-trip safety).

### В IDE

Настройте «форматировать при сохранении» через хук на команду
`yasny формат -в %f` для текущего файла.

## Проверка кода (линтер)

Команда `yasny проверить` выполняет статический анализ — находит
потенциальные проблемы без запуска программы.

### Использование

```
yasny проверить файл.ya    — проверить один файл
yasny проверить .          — все .ya в текущей папке (рекурсивно)
yasny проверить проект/    — все .ya в указанной папке
```

### Правила

| Код | Что находит |
|---|---|
| `unused-var` | Неиспользуемые переменные, константы, импорты |
| `unused-func` | Объявленные но не вызываемые функции |
| `unreachable` | Код после `вернуть`, `бросить`, `прервать` |
| `shadow` | Затенение переменной из внешнего скоупа |
| `empty-block` | Пустые блоки `если`/`для` |
| `const-reassign` | Попытка изменить константу |

### Пример

```
$ yasny проверить main.ya
⚠ main.ya:5 [unused-var] константа 'х' объявлена но не используется
⚠ main.ya:12 [unreachable] недостижимый код после вернуть/бросить/прервать

Найдено проблем: 2
```

### Подавление предупреждений

Переменные с префиксом `_` игнорируются линтером:

```
конст _временная = вычислить()  # линтер не предупредит
```

## Часто встречающиеся проблемы

### «Идентификатор не найден» при использовании `пусть`

`пусть` удалён в версии 0.5. Используйте `конст` для неизменяемых
переменных и `перем` для изменяемых.

### «Идентификатор не найден» при использовании `возврат`

Каноническая форма — `вернуть` (глагол повелительного наклонения).

### `//` теперь не комментарий

Только `#`. Если в редакторе поставлен `//`, замените на `#`.

### `json_создать` или `regex_*` не работают

Они теперь модули: `json.создать`, `регвыр.найти`,
`регвыр.заменить`, `регвыр.совпадает`.

### Интерполяция не сработала

Интерполяция `{...}` работает только в двойных кавычках. В одинарных
строки берутся буквально.

```
вывод("Сумма: {2+2}")     # Сумма: 4
вывод('Сумма: {2+2}')     # Сумма: {2+2}
```

### Деление двух целых даёт дробь

Деление `/` всегда возвращает дробное число, если результат не целый:
`7 / 2 == 3.5`. Для целочисленного деления используйте `//`:

```
вывод(7 // 2)      # 3
вывод(7 % 2)       # 1
```

### Сравнение объектов

В текущей версии равенство объектов и массивов — по содержимому. Для
сложных структур надёжнее сравнивать через `json.создать`.

## Известные ограничения версии 0.5

- **Множественный возврат `вернуть a, b` работает**, но только в
  блочной форме функции, не в inline. Корректно:
  ```
  функция дивмод(a, b)
      вернуть a // b, a % b
  конец
  конст ч, о = дивмод(17, 5)
  ```
  Не работает: `функция дивмод(a, b): a // b, a % b`.

- **`повтори N раз` не реализован.** Используйте `для i от 1 до N`.

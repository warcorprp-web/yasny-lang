# Ясный

Язык программирования с русскоязычным синтаксисом. Версия 0.5.

```
конст имя = "мир"
вывод("Привет, {имя}!")
```

## Зачем нужен Ясный

Когда новичок без знания английского начинает учить программирование,
его внимание делится между алгоритмом и переводом ключевых слов. Что
такое `def`, `return`, `for in`, как читается `if-elif-else` — всё
это лишний слой между человеком и сутью того, что он пишет.

Ясный убирает этот слой. На нём можно изучить переменные, условия,
циклы, функции, классы — на родном языке. После этого переход на
Python или JavaScript занимает дни, а не месяцы: концепции уже
понятны, остаётся только запомнить английские слова.

Ясный не заменяет Python или JS. Это первый шаг, мост.

## Установка

Понадобится Go 1.22 или новее.

```
git clone https://github.com/warcorprp-web/yasny-lang.git
cd yasny-lang
go build -o yasny .
```

После сборки положите бинарник `yasny` в `$PATH`.

## Запуск

```
yasny файл.ya          # выполнить файл
yasny запустить        # выполнить точку входа из пакет.json
yasny                  # интерактивный режим (REPL)
```

## Пакеты

Ясный использует git/GitHub как источник пакетов. Реестр коротких
имён хранится в [отдельном репозитории](https://github.com/warcorprp-web/yasny-registry).

Самый быстрый сценарий — один файл и пакет:

```
echo 'импорт м из "матем"
вывод(м.факториал(5))' > программа.ya

yasny подключить матем     # пакет.json создаётся автоматически
yasny программа.ya         # 120
```

Команды:

```
yasny инит мой_проект                      # явная инициализация
yasny подключить матем                      # короткое имя из реестра
yasny подключить github.com/у/п@v1.0.0     # полный URL и тег
yasny подключить                            # всё из пакет.json + пакет.lock
yasny список                                # установленные пакеты
yasny запустить                             # запустить точку входа
```

Подробнее — в [РУКОВОДСТВО.md](РУКОВОДСТВО.md#пакеты).

## Форматирование кода

Команда `yasny формат` приводит код к единому канону, описанному в
[СТИЛЬ.md](СТИЛЬ.md):

```
yasny формат файл.ya       # вывести в stdout
yasny формат -в файл.ya    # переписать файл на месте
yasny формат -в .          # все .ya в проекте (рекурсивно)
```

Форматер сохраняет комментарии, пустые строки, inline-формы, лямбды,
pipeline и декораторы — он причёсывает только пробелы, отступы и
пунктуацию. Поведение программы после форматирования остаётся
прежним.

## Проверка кода (линтер)

Команда `yasny проверить` находит потенциальные проблемы без запуска
программы:

```
yasny проверить файл.ya    # проверить один файл
yasny проверить .          # все .ya в проекте (рекурсивно)
```

Что находит:
- Неиспользуемые переменные, константы, импорты
- Неиспользуемые функции
- Недостижимый код после `вернуть`/`бросить`/`прервать`
- Затенение переменных во вложенных скоупах
- Пустые блоки `если`/`для`
- Попытка изменить константу

## Основы языка

### Переменные

```
конст имя = "Иван"          # неизменяемая
перем счётчик = 0           # изменяемая
счётчик = счётчик + 1
счётчик += 5
```

`конст` — основная форма. `перем` используется только тогда, когда
переменную действительно нужно менять.

### Типы данных

```
конст число = 42
конст дробь = 3.14
конст текст = "строка"
конст флаг = да              # или нет
конст список = [1, 2, 3]
конст словарь = {"ключ": "значение"}
конст пусто = ничего         # null-значение
```

### Интерполяция строк

В двойных кавычках можно вставлять выражения в `{...}`:

```
конст имя = "Аня"
вывод("Привет, {имя}!")
вывод("2 + 2 = {2 + 2}")
```

В одинарных кавычках интерполяция не работает — это обычная строка.

### Условия

```
если возраст >= 18
    вывод("Взрослый")
иначеесли возраст >= 14
    вывод("Подросток")
иначе
    вывод("Ребёнок")
конец
```

Короткая форма для одного выражения:

```
если возраст >= 18: вывод("Взрослый") иначе: вывод("Ребёнок")
```

### Циклы

Со счётчиком:

```
для i от 1 до 10
    вывод(i)
конец

для i от 0 до 100 по 5    # с шагом 5
    вывод(i)
конец
```

По коллекции:

```
для имя в ["Аня", "Боря", "Вася"]
    вывод(имя)
конец
```

С условием:

```
перем k = 0
пока k < 5
    вывод(k)
    k += 1
конец
```

### Функции

```
функция приветствовать(имя)
    вернуть "Привет, {имя}!"
конец

вывод(приветствовать("Маша"))
```

Параметр по умолчанию:

```
функция степень(основание, показатель = 2)
    вернуть основание * основание
конец
```

Короткая форма с автоматическим возвратом значения:

```
функция квадрат(x): x * x
```

Лямбда:

```
конст удвоить = x => x * 2
```

### Классы

```
класс Точка
    функция создать(x, y)
        это.x = x
        это.y = y
    конец

    функция расстояние_до(другая)
        конст dx = это.x - другая.x
        конст dy = это.y - другая.y
        вернуть корень(dx * dx + dy * dy)
    конец
конец

конст a = Точка(0, 0)
конст b = Точка(3, 4)
вывод(a.расстояние_до(b))
```

Наследование:

```
класс Животное
    функция создать(имя)
        это.имя = имя
    конец

    функция говорить()
        вернуть "{это.имя} издаёт звук"
    конец
конец

класс Кот наследует Животное
    функция говорить()
        вернуть "{это.имя} мяукает"
    конец
конец

конст барсик = Кот("Барсик")
вывод(барсик.говорить())
```

`создать` — имя конструктора. Если в потомке его не объявить, будет
использован конструктор родителя.

### Сопоставление с образцом

```
конст имя = совпадение день
    когда 1: "Понедельник"
    когда 2: "Вторник"
    когда 3: "Среда"
    иначе: "Ещё какой-то день"
конец
```

### Обработка ошибок

```
попытка
    рискованная_операция()
поймать ошибка
    вывод("Не получилось: {ошибка}")
всегда
    очистить()
конец
```

Бросить ошибку:

```
если делитель == 0
    бросить "деление на ноль"
конец
```

### Тесты

Тесты — часть языка, отдельные библиотеки не нужны:

```
тест "сложение работает"
    проверить плюс(2, 3) == 5
    проверить плюс(-1, 1) == 0
конец
```

Запустите файл, чтобы увидеть результат каждого теста.

## Стандартная библиотека

### Вывод и ввод

- `вывод(значение)` — вывести в консоль
- `ввод(приглашение)` — прочитать ввод пользователя

### Преобразования

- `тип(x)` — тип значения
- `строка(x)`, `число(x)` — преобразование

### Числа и строки

- `длина(x)` — длина массива или строки
- `мин(a, b)`, `макс(a, b)`, `абс(x)`, `округл(x)`, `корень(x)`,
  `степень(основание, показатель)`

### Массивы

- `добавить`, `удалить`, `вставить`
- `фильтр`, `преобразовать`, `свернуть`, `найти`, `найти_индекс`
- `сортировать`, `реверс`, `объединить`, `соединить`
- `сумма`, `все`, `любой`, `взять`, `пропустить`
- `уникальные`, `объединение`, `пересечение`, `разность`

### Словари

- `ключи(объект)`, `значения(объект)`, `получить(объект, ключ, по_умолчанию)`

### Строки

- `верхний`, `нижний`, `обрезать`, `повторить`
- `разделить`, `соединить`, `заменить`, `подстрока`
- `содержит`, `начинается_с`, `заканчивается_на`

### Файлы

- `читать_файл(путь)`, `записать_файл(путь, содержимое)`,
  `существует_файл(путь)`

### Модули, доступные без импорта

- `мат` — числа и тригонометрия: `мат.пи`, `мат.е`, `мат.sin`,
  `мат.cos`, `мат.лог`, `мат.пол`, `мат.потолок`
- `время` — дата и время: `время.сейчас()`, `время.строка()`,
  `время.год()`, `время.спать(мс)`, `время.разобрать(строка, формат)`
- `json` — `json.создать(объект)`, `json.разобрать(строка)`
- `регвыр` — регулярные выражения: `регвыр.найти`, `регвыр.заменить`,
  `регвыр.совпадает`
- `http` — HTTP-клиент и сервер: `http.получить/пост/положить/удалить(url)`,
  `http.приложение()` → routing, middleware, JSON body, статика
- `крипто` — хеши и шифрование: `sha256/sha1/md5`,
  `hmac_sha256/hmac_sha1(сообщение, ключ)`,
  `aes_зашифровать/aes_расшифровать(текст, ключ)`,
  `jwt_создать(данные, секрет)`, `jwt_проверить(токен, секрет)`,
  `base64/hex кодировать/декодировать`
- `бд` — базы данных (SQLite + PostgreSQL): `бд.открыть(путь)` для SQLite,
  `бд.подключить("postgres://...")` для PostgreSQL. Единый API:
  `соед.выполнить/запрос/строка/значение(sql, ...)`,
  `соед.подготовить(sql)`, `соед.вставить_много(таблица, колонки, данные)`,
  `соед.транзакция(функция(тх) ...)`, `соед.мигрировать(версия, sql)`,
  `соед.таблицы()`, `соед.колонки(таблица)`, `соед.версия()`
- `csv` — табличные данные: `csv.разобрать(текст)`,
  `csv.с_заголовками(текст)`, `csv.строка(массив)`
- `случайное` — `случайное.число()` (0..1),
  `случайное.целое(мин, макс)`, `случайное.элемент(массив)`
- `файлы` — `файлы.читать(путь)`, `файлы.записать(путь, содержимое)`,
  `файлы.существует(путь)` (то же что `читать_файл` и т.п.)
- `путь` — манипуляции путями: `путь.соединить`,
  `путь.имя_файла`, `путь.расширение`, `путь.абсолютный`
- `ос` — `ос.переменная_среды(имя)`, `ос.аргументы()`, `ос.выйти(код)`
- `cli` — парсинг аргументов командной строки
- `шаблон` — шаблонизатор HTML/текста: `шаблон.рендер(строка, данные)`,
  `шаблон.файл(путь, данные)`. Синтаксис: `{{выражение}}`,
  `{{если ...}}`, `{{для x в массив}}`, `{{включить "файл"}}`
- `вс` — WebSocket: `вс.подключить(url)` (клиент),
  `вс.сервер(порт, обработчик)` (сервер),
  `сокет.отправить/получить/слушать/закрыть`

## Переход на Python и JavaScript

Все концепции переносятся один в один — меняются только ключевые
слова.

| Ясный            | Python              | JavaScript          |
|------------------|---------------------|---------------------|
| `конст x = 5`    | `x = 5`             | `const x = 5`       |
| `перем x = 5`    | `x = 5`             | `let x = 5`         |
| `функция f()`    | `def f():`          | `function f() {}`   |
| `вернуть x`      | `return x`          | `return x;`         |
| `если ... иначе` | `if ... else`       | `if ... else`       |
| `пока`           | `while`             | `while`             |
| `для x в y`      | `for x in y`        | `for (x of y)`      |
| `класс`          | `class`             | `class`             |
| `создать`        | `__init__`          | `constructor`       |
| `это`            | `self`              | `this`              |
| `да` / `нет`     | `True` / `False`    | `true` / `false`    |
| `пусто`          | `None`              | `null`              |
| `выдать`         | `yield`             | `yield`             |
| `асинх` / `ждать`| `async` / `await`   | `async` / `await`   |
| `конец`          | (отступы)           | `}`                 |

После этой таблицы и недели практики на Python/JS вы пишете на новом
языке без затруднений.

## Принципы языка

См. [PRINCIPLES.md](PRINCIPLES.md). Главное:

- Программа читается как алгоритм на русском.
- Слова важнее символов.
- Один способ сделать одно дело.
- Иммутабельность по умолчанию.
- Ошибки с подсказками.
